A VS Code extension (internal name "codegraph") that builds a Code Property Graph (CPG) of a workspace and stores it in FalkorDB. The CPG is used for taint analysis (finding data flow from sources to sinks across files).

**Why:** Security analysis — find vulnerabilities like SQL injection, XSS, etc. by tracking data flow through a codebase graph.

**How to apply:** All suggested changes should serve the goal of accurate taint tracking. When in doubt, favor completeness of data flow edges over terseness of the graph.

## Stack
- TypeScript + VS Code Extension API
- Tree-sitter (native Node add-on) for CST parsing
- FalkorDB (graph database) for storage
- Bun as test runner (bun test), tsc for compilation
- Joern CPG specification as the node/edge schema

## CPG Pipeline (3 layers, all implemented)
1. **UAST** (`src/graph/cpg/uast/`) — CST → Joern node types via language adapters
2. **CFG** (`src/graph/cpg/cfg/`) — Control flow edges between UAST METHOD/BLOCK/CONTROL_STRUCTURE nodes
3. **PDG** (`src/graph/cpg/pdg/`) — REACHING_DEF and CDG edges for data flow

## Language Support
- TypeScript/JavaScript: fully adapted (`TypeScriptAdapter.ts`, 48 CST type mappings)
- Python: adapter added session 1 (`PythonAdapter.ts`, ~24 mappings)
- Go/Rust/Java/JavaScript: CST snapshots exist but no adapters yet

## Key Directories
- `src/graph/cpg/uast/adapters/` — one adapter file per language
- `test/fixtures/code_examples/{language}/` — hello_world fixtures for each language
- `test/__snapshots__/tree-sitter-cst/{language}/` — raw CST JSON snapshots (generated by `npm run test:update-goldens`)
- `test/__snapshots__/uast/{language}/` — UAST JSON snapshots (generated by `bun run scripts/generate-uast-snapshot.ts`)
- `scripts/` — standalone Bun scripts for dev tooling (not compiled by tsc)
- `design/02_planned/CPG/` — design docs for UAST, CFG, PDG layers
