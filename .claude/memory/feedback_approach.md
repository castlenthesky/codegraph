"START SIMPLE" is an explicit signal to not over-engineer. When the user says this, implement the minimum viable thing that works — don't add abstractions, configurability, or future-proofing beyond what the task requires.

**Why:** Stated directly — keep scope tight, audit output, then iterate.

**How to apply:** When asked to add a feature, ask "what's the smallest thing that proves this works?" and do that. Don't add generic infrastructure for hypothetical future languages/cases until there's a second concrete use case that demands it.
